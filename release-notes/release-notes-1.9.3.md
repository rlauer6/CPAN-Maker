# CPAN::Maker 1.9.3

## Bug Fixes

### `_generate_man_links` — incorrect symlink source path

The `Makefile.PL` postamble generated by `_generate_man_links` was
using `$(DESTINSTALLMAN3DIR)` as the symlink source in the `ln -sf`
call and as the path recorded in the `.packlist`. Both should
reference
`$(INSTALLMAN3DIR)` — the real installed location — rather than the
staging destination used during `make install`. The incorrect source
caused broken symlinks after installation and incorrect `.packlist`
entries that would confuse `ExtUtils::Installed` and uninstall
tooling.

### `main` — modulino exit code handling

`main` previously called `exit` internally, preventing it from being
called as a library function or tested without terminating the
process.
`main` now returns the exit code from `$cli->run()` and the top-level
modulino guard handles the `exit`: `caller or exit __PACKAGE__->main()`.
